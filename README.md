# No Reaction

高中化学主题的双人离子卡牌对战游戏。玩家轮流向公共反应区打出离子牌，目标是**不让反应区发生反应**。迫使沉淀或气体反应发生、且无法用道具解除的玩家判负。

纯 HTML/CSS/JS 实现，浏览器直接打开即可玩本地双人；起一个零依赖的本地服务器后，支持 6 位房间码联机对战（局域网 / 樱花 frp 远程，免费、无需注册任何账号）。

---

## 快速开始

### 本地双人（最简单）

直接用浏览器打开 `index.html`，点「本地双人（同屏热座）」即可开玩，无需任何环境。

### 起本地服务器（推荐，联机必需）

```bash
node server.js
# 然后浏览器打开 http://localhost:5173
```

或：

```bash
npm run serve
```

## 玩法规则

### 目标
保持反应区「无反应」（No Reaction）。当你打出的离子与反应区已有离子发生**沉淀或气体反应**，且用尽道具仍无法解除时，你判负。

### 回合流程
1. **摸牌** — 从牌堆摸 1 张（先手首回合不摸，抵消先手优势）
2. **出牌** — 打出 1 张离子牌到反应区
3. **反应判定** — 与区内所有离子逐一检测
   - **中和**（H⁺ + OH⁻）：自动成对移入弃牌区，不判负，回合继续
   - **沉淀 / 气体**：进入道具阶段
4. **道具阶段** — 使用道具解除反应（可连用多张），或认输该反应
5. **判负** — 道具用尽仍有反应未解除 → 判负

### 道具牌
| 道具 | 类型 | 效果 |
|------|------|------|
| 过滤 | 防守 | 移走一对沉淀离子（阳 + 阴） |
| 加热 | 防守 | 移走一个参与气体反应的离子 |
| 萃取 | 防守 | 把反应区指定离子取回手牌，任意反应可用 |
| 中和 | 防守 | 移走一个 H⁺ 或 OH⁻（不能处理 NH₄⁺ + OH⁻） |
| 搅拌 | 中性 | 清空整个反应区到弃牌堆 |
| 催化剂 | 攻击 | 无反应时打出，双方各出一张离子后统一判定 |

完整规则与反应判定表见 [DESIGN.md](DESIGN.md)。

```
no-reaction/
├── index.html          # 首页：本地双人 / 创建 / 加入房间

├── DESIGN.md           # 完整设计文档
├── config/
│   └── cards.js        # 牌的数量与离子/道具定义（集中配置）
├── css/
│   └── style.css       # 全局样式
├── js/
│   ├── reactions.js    # 离子反应判定表（中和/沉淀/气体）
│   ├── engine.js       # 核心游戏引擎（纯逻辑，无 DOM 依赖）
│   ├── network.js      # 联机层（与自托管中继服务器通信）
│   ├── home.js         # 首页控制器
│   └── ui.js           # 游戏页控制器
└── test/
    ├── engine.test.js  # 引擎单元测试
    └── sim.test.js     # 随机对局模拟测试
```

### 调整卡牌数量
所有离子/道具的数量都在 `config/cards.js` 集中管理，改 `count` 字段即可，引擎与 UI 自动读取。反应判定表在 `js/reactions.js`，可独立增改。

## 联机对战

联机由内置的中继服务器（`server.js`）实现，**不需要注册账号、不依赖任何第三方服务、零依赖**。只要页面是通过 `node server.js` 起的服务打开的（而不是直接双击 html），联机就自动可用，无需任何配置。

服务器只当「哑中继」：在内存里存房间数据，并通过 SSE 把变化实时推给两个玩家；游戏规则仍由房主权威结算。

### 玩法一：局域网联机（同一 WiFi）

最简单。一台机器起服，其他人连它的内网地址。

1. 房主电脑运行 `node server.js`。
2. 查本机内网 IP（Windows：`ipconfig` 看 IPv4，形如 `192.168.x.x`）。
3. 同一 WiFi 下的其他人，浏览器打开 `http://<那个IP>:5173`。
4. 一人「创建房间」拿到 6 位房间码，另一人「加入房间」输入即可。

### 玩法二：远程联机（樱花 frp）

跨网络也能玩，免费。原理是用 frp 把本机的 5173 端口映射到一个公网地址。

1. 房主电脑运行 `node server.js`。
2. 在[樱花 frp](https://www.natfrp.com/) 注册，新建一条 **TCP / HTTP 隧道**，本地端口填 `5173`，启动客户端。
3. 樱花会给你一个公网地址（形如 `xxx.natfrp.cloud:12345`）。
4. 把这个公网地址发给对方，双方都用它打开页面，创建/加入房间即可。

> 提示：建 HTTP 类型隧道访问体验更好；建 TCP 隧道则用 `http://公网地址:端口` 访问。
> 房间无鉴权，拿到房间码即可加入，适合朋友间对战，别传敏感信息。

同步策略：房主（host）是权威端，持有完整状态并运行引擎；访客（guest）只提交「动作意图」，由房主校验后整份回写状态。规则只在一处执行，避免双端不一致。

## 技术说明

- **零构建**：原生 HTML/CSS/JS，无需打包工具。
- **引擎可移植**：`engine.js` / `reactions.js` / `cards.js` 同时支持浏览器（`window.*`）与 Node（`module.exports`），因此测试可在 Node 直接跑。
- **可复现随机**：引擎用可注入种子的 `mulberry32` 洗牌，便于测试复现与联机同步。
